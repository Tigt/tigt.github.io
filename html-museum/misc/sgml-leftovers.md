> SGML wants a hierarchy. But we need to be able to mark “the passage that seems to echo Vergil” and “the passage the beer spilled on” at the same time.

To this day, we haven't solved this issue. The least-bad solution seems like an element that would take an XPath range selector, and that's plenty bad. [This 1995 email](http://ksi.cpsc.ucalgary.ca/archives/HTML-WG/html-wg-95q2.messages/0271.html) proposes basically that, but with regular expressions instead.
